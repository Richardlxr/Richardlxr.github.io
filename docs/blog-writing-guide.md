# 博客长文写作指南

这份指南分成两部分：网站会自动帮你做什么，以及你作为作者写 Markdown 时最好注意什么。

## 网站自动做的事

### 1. 自动生成文章页

你只要把 `.md` 放进：

```text
src/content/posts/
```

Astro 会把它变成对应网页。

```text
src/content/posts/my-note.md -> /writing/my-note/
```

### 2. 自动生成文章列表

首页和 `/writing/` 会自动读取 Markdown 文章。你不需要手动改文章列表。

### 3. 自动生成目录

文章里的二级、三级标题会自动变成目录。

```md
## 第一节

### 一个小问题
```

目录会显示在文章开头，帮助读者先看到结构。

### 4. 自动优化 Markdown 样式

正文、标题、引用、列表、代码块、图片都会套用统一样式。你不用自己写 HTML。

## 作者写作时要注意的事

### 1. 正文小节从 `##` 开始

文章页面已经有一个大标题，所以正文里不要再用 `#`。

推荐：

```md
## 工程：程序员存在的原因
```

不推荐：

```md
# 工程：程序员存在的原因
```

### 2. 每 4 到 6 段给读者一个停顿

长文最怕一路都是普通段落。可以穿插：

- 一个小标题
- 一句引用
- 一个列表
- 一张图片
- 一段代码或数据

### 3. 重点句用引用块

适合放文章里最想让人记住的话。

```md
> 工程要的，不只是代码正确，而是最后功能真的完成。
```

### 4. 多个原因或现象用列表

不要把所有信息塞进一个长段落。

```md
- 地面摩擦不均
- 电量变化
- 电机响应存在差异
- 传感器读数有噪声
```

### 5. 控制段落长度

普通段落尽量控制在 150 到 250 个中文字符。特别长的段落可以拆成两段。

### 6. 一篇文章最好有清楚的骨架

写之前可以先列这个骨架：

```md
## 我遇到了什么问题

## 我一开始怎么理解它

## 真实测试暴露了什么

## 我最后怎么想
```

这样文章会更像一条路，而不是一整块文字。

## 推荐模板

```md
---
title: "文章标题"
description: "一句话说明这篇文章讲什么。"
date: "2026-06-14"
tags:
  - 学习
  - Blog
---

## 开头：我为什么写这篇

这里写背景。

> 放一句最核心的判断，给读者一个抓手。

## 中间：事情是怎么展开的

这里写过程。

- 第一个关键点
- 第二个关键点
- 第三个关键点

## 结尾：我现在怎么看

这里写总结和反思。
```
